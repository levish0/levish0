## Flow Chart VS PseudoCode
우리는 코드를 짜기 전에, 생각을 해 전체적 흐름이나 기능들을 정해놓고 시작해야 한다. 이를 위해 사람들이 사용하는 방식이 **Flow chart**와 **PseudoCode**이다.

|Flow Chart |PseudoCode |
|:---:|:---:|
|![flow_chart](https://github.com/shiueo/shiueo/blob/main/web/public/flow_chart.jpg?raw=true) from [StackOverflow](https://stackoverflow.com)| ![pseudocode](https://github.com/shiueo/shiueo/blob/main/web/public/pseudocode.png?raw=true) from [computerscience.gcse.guru](https://www.computerscience.gcse.guru)|

여기서 확인할 수 있듯이 **Flow Chart**는 간단한 도형들을 사용해 시각적으로 프로그램의 전체적 흐름을 나타내는 것이고, **PseudoCode**는 사람이 알아볼 수 있게만, 대략적인 과정을 중간언어 쯤으로 나타내어 놓는 것이다. ~~솔직히 Python은 너무 직관적이라 PseudoCode로 써도 아무 문제가 없다.~~


## PseudoCode
- 영어로 작성
- 체계적으로 구성되어야 함
- 어떤 프로그램인지 이름이 명시되어야 함
- Pre-condition, Post-condition이 정의되어야 함
- Return value가 표시되어야 함 (결과)

다음 조건들을 지킨다면 잘 작성된 Pseudocode라고 할 수 있다.

결국 PseudoCode를 작성한다는 것은 어떤 문제를 풀기 위한 알고리즘의 초안을 작성하는 것과 마찬가지이므로, 이 알고리즘에서 중요한 것도 몇가지 집자면

- Correctness
- Amount of work done
- Amount of space used
- Simplicity & Clarity

가 있다.

## Examples
예시로 몇몇개를 작성해보도록 하겠다.
1~N까지 더하는 프로그램을 작성한다고 해보자. ~~N(N+1)/2는 제외한다.~~

```python
Algorithm Sum_N
Pre-condition: N
Post-condition: print sum of 1~N

read number N
for i in 1~N:
    sum += i

return sum
```
이런 식이다. 위 코드는 완전히 틀렸고 어디에 넣더라도 동작하지 않겠지만, 인간이 이해하는데에는 아무 문제가 없다.
